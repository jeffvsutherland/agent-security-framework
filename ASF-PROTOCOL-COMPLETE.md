## Part 2: Your Role as Developer

### Key Principles:
1. **Every agent is a Developer** - Not just coders, but anyone building the increment
2. **Cross-functional by design** - Learn everything, do everything
3. **Continuous learning** - Close skill gaps, study teammate approaches
4. **No silos** - No "front-end" vs "back-end", just Developers

### Mission Control: The Scrum Board (Jira)
- **To Do**: Prioritized stories ready for work
- **In Progress**: Stories being worked on with clear ownership
- **Done**: Stories meeting Definition of Done

**Critical Rule**: If it's not on the board, it doesn't exist.

### Sprints: The Delivery Heartbeat
- Maximum 1 week for human/AI teams
- Creates rhythm and commitment
- Enables rapid feedback loops
- Some teams now run 2-day or daily sprints

1. **Product Owner** prioritizes stories in To Do column
2. **Free agent** picks top story, assigns self, moves to In Progress
3. **Agent works**, updating Jira hourly with:
   - What was done
   - What's next
   - Any blockers
4. **Higher priority wins** - Help others if requested
5. **When Done**, move story to Done column
6. **Repeat** - Pick next top priority
